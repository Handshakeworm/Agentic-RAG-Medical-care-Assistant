# Chunk Enrichment 生成规范

## Tags 生成规范

标签应覆盖以下维度（按适用性选取，不强制全覆盖）：

| 维度 | 示例 |
|------|------|
| 疾病名称 | 心力衰竭、2型糖尿病、急性心肌梗死 |
| 涉及科室 | 心内科、内分泌科、急诊科 |
| 治疗方式 | 药物治疗、手术治疗、介入治疗 |
| 检查手段 | 心脏超声、血常规、CT |
| 药物类别 | β受体阻滞剂、ACEI/ARB、利尿剂 |
| 人群特征 | 老年患者、妊娠期、儿童 |

### 标签粒度要求

- 优先使用最具体的术语：用"射血分数降低型心衰"而非"心脏病"
- 同时包含上位概念和下位概念：["心力衰竭", "HFrEF"]
- 不超过 5 个标签，避免标签膨胀

## Hypothetical Questions 生成规范

### 目标

弥合患者口语描述与临床术语之间的语义鸿沟。

### 口语化转换示例

| 临床术语 | 患者口语 |
|---------|---------|
| 心力衰竭急性失代偿 | "最近突然喘不上气，躺着更严重" |
| 房颤伴快速心室率 | "心跳特别快还不规律，感觉心脏乱跳" |
| 低钾血症 | "总觉得浑身没劲，腿软" |
| 药物不良反应：ACEI 干咳 | "吃了降压药以后老是咳嗽，停药就好了" |

### 生成原则

1. 必须以第一人称患者视角撰写
2. 使用日常用语，不出现任何医学术语
3. 包含具体的感受描述（疼痛、不适的具体感觉）
4. 可以包含时间、场景等生活化细节
