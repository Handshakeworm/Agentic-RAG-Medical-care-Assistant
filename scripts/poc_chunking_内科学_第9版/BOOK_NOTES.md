## 2. 章节结构 — 第一次出现 L4 字典层级

跟前两本最大差异:**TOC 把节下"一、xxx"和"[附] xxx" 作为独立目录条目列出**(共 107 条),字典必须收为 L4。

| 层级 | anchor | 数量 | 例 |
|---|---|---|---|
| L1 | 第N篇 | 9 | 第二篇 呼吸系统疾病 |
| L2 | 第N章 | 137 | 第六章 肺部感染性疾病 |
| L3 | 第N节 | 166 | 第二节 细菌性肺炎 |
| **L4** | `^[一二...]、` 或 `^\[附` | **107** | 一、肺炎链球菌肺炎 / [附] 流行性感冒 |

**对 Step 1 的影响**:
- `PATTERNS` 加 2 条 L4 pattern(中文数字+顿号 / `[附]`)
- `_update_stack` 槽位扩到 5
- `lookup` 收到 L4(原 SOP 只到 L3)

**对 Step 3 的影响**(关键):
- L4 已经成 section 边界,Pass 1 不能再用 一、 切(会重复)
- Pass 顺序降级:**Pass 1 = 【】(原 (一)),Pass 2 = (一)(原 1.)**

**mixed depth 同样存在**:第一篇 绪论 → 直接挂 L4(无章无节);其他篇 → 章 → 节 → L4。

## 3. 硬编码补丁 — 算法救不了的 OCR 错(新增方法)

mineru 在本书 TOC 提取出 6 类硬错,**算法层面救不了**(单一行漏识/章号 OCR 错/特殊字符残留),走人工核对 PDF + 硬编码补丁。

实现:`poc_build_toc_dict.py` 里 `PATCH_REPLACE_TITLE` + `PATCH_INSERT_AFTER` + `_apply_patches` + `_rebuild_paths`。

### 3.1 补丁类型

| 类型 | 数量 | 示例 |
|---|---|---|
| **REPLACE_TITLE**(改 title)| 3 | `第二十三章 糖尿病` → `第二十二章 糖尿病`(OCR 二/三错)/ `第十章 肺血栓栓寒症` → `肺血栓栓塞症`(OCR 寒/塞错)/ `三、XX 男性综合征 •` → 去末尾 `•` |
| **INSERT_AFTER**(在锚点后插 entry)| 6 | 第六章溶血性贫血 第四节血红蛋白病下加 `一、珠蛋白生成障碍性贫血` / `二、异常血红蛋白病`(mineru 把 3 行黏成 1 paragraph)/ 第五篇第六章肾小管疾病 第一节下加 `三、混合性肾小管酸中毒`(漏行)/ 第三篇第四章第三节下加 `三、缺血性心肌病`(OCR 漏顿号 → "三 缺血性心肌病")/ 第九篇第二章中毒下加 `第一节 概述`(漏行)/ 第七篇第二十六章下加 `第一节 水、钠代谢失常`(漏行)|
| **同篇章号断序救** | 1 | "第二十三章 糖尿病" + "第二十三章 低血糖症" 同号 → 前者改第二十二章 + path 同步修 |

### 3.2 `_rebuild_paths` 必做

INSERT_AFTER 后,如果新插的是 L3 节,**下游 L4 entries 的 heading_path 还挂在 L2 章下**(因为原来没 L3)。`_apply_patches` 末尾必须走一遍 `_rebuild_paths` (stack-walk 重算所有 path),把 L4 自动重新挂到新 L3 下。

例:补"第一节 水、钠代谢失常"前,`一、失水` path 是 `第七篇 / 第二十六章 / 一、失水`(缺中间 L3);补后 path 自动变成 `第七篇 / 第二十六章 / 第一节 水、钠代谢失常 / 一、失水`。

### 3.3 PDF 截图人审工作流

发现 OCR 错的标准做法:
1. 跑 Step 1 出完整 tree
2. 用户给 PDF TOC 截图(全部目录页)
3. 我 vs 字典逐篇对比,标 ⚠ 真错 / ✓ 接受(原书印刷如此)
4. 真错部分写硬编码 patch
5. 重跑 Step 1 → spot-check 关键 path

## 4. mineru 在本书的孤立 OCR bug(新发现)

| bug | 现象 | 应对 |
|---|---|---|
| **`(N)` 形式被错识为篇** | `第四篇 消化系统疾病` OCR 成 `(4) 消化系统疾病`(失"第N篇"前缀)| `_classify` 先跑 `PIAN_PAREN_RE`:`^[\(（]\s*([1-9])\s*[\)）]\s+(\S{2,})` 命中后改成 `第{中文数字}篇 X`,再走 PATTERNS L1 |
| **同篇内章号重复(OCR 二/三混淆)** | 第七篇下 "第二十二章 糖尿病" 错识"第二十三章" → 跟"第二十三章 低血糖症"撞号 | 见 §3.1 硬编码 |
| **漏识整行**(L3 节 / L4 一、)| 见 §3.1 INSERT_AFTER 6 处 | 硬编码 |

注:bug "(N) → 第N篇" 仅本书出现 1 次(第四篇),但 PIAN_PAREN_RE 是通用救手段,可下沉到 SOP。

## 6. Pass 策略(渐进 2 pass + 跨 section 吸收)

```python
PARENT_SPLIT_THRESHOLD = 5000      # > 5000 触发 Pass 1
PARENT_REFINE_THRESHOLD = 6000     # > 6000 触发 Pass 2
CHAPTER_ABSORB_THRESHOLD = 500     # L1/L2/L3 < 500 → 跨 section 吸收(L4 不参与)

# Pass 1: 【】 (无条件,> 5000 触发)— 本书主力,1680 个候选
# Pass 2: (一)(二) (条件,> 6000 触发)— 救节内还有 (一) 的极少数
# Pass 3: 不启用(粒度过细)
```

**为什么 ABSORB 扩到 L3**:本书字典扩到 L4,L3 节内容被 L4 切走后,节级常只剩短引言/附引(< 500 字)。L1/L2/L3 都允许吸收,L4 永远保留(L4 是最小语义单元,即使 28 字交叉引用 stub 也独立)。

## 8. 节末"参考文献"标识 — 用"推荐阅读"

本书**没有**节末"参考文献"标题(全 0),但**每篇末有"推荐阅读"+ 1 段英文/中文 reference list**(8 处)。

通用 `RE_REF_MARKER` 扩展为同时匹配两者:

```python
RE_REF_MARKER = re.compile(r"^(?:参考文献|推荐阅读)\s*$")
```

效果:8 处推荐阅读 + 后续 list block 全部走 `ref_dropped` 路径丢弃,共 24 blocks / 11,083 字符。**消掉 3 个 > 2000 字超大子块**(原 max 3152 → 1287)。
